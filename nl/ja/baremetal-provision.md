# カスタム・ベアメタル・サーバーの作成
{: #ordering-baremetal-server}

カスタム {{site.data.keyword.baremetal_short}} を作成するには、以下のステップを使用します。

1. [{{site.data.keyword.cloud_notm}} カタログ](https://console.bluemix.net/catalog/){:target="_blank"}を開きます。   
2. 「ベアメタル・サーバー」を選択します。
3. 「作成」をクリックします。
4. サーバー・リストの下で、次のステートメントを探します。**「他の構成オプションを表示するには、ここをクリックしてください (Interested in other configuration options? Click here)」**。このオプションを選択します。カスタム・サーバー・フォームが表示されます。
1. サーバーのデータ・センター・ロケーションを選択します。
* **「開始価格 (Starting Price Per)」** リンクをクリックして、サーバーの 3 つのカテゴリーからサーバーを選択します。
  * SAP 認定サーバー (SAP 認定サーバーのプロビジョニングについて詳しくは、『[{{site.data.keyword.cloud_notm}} SAP 認定インフラストラクチャー](/docs/bare-metal/bare-metal-sap-applications.html)』を参照してください)
  * シングル・プロセッサー・マルチ・コア・サーバー
  * デュアル・プロセッサー・マルチ・コア・サーバー

* 構成オプションから選択します。**「データ・センター」**、**「RAM」**、および **「オペレーティング・システム」**は必須フィールドです。他のフィールドはすべてオプションです。オプションのフィールドについて詳しくは、『**[追加のサーバー構成オプション](#addl-server-options)**』を参照してください。

    **注**: サーバーとオペレーティング・システムの間に矛盾が存在する場合は、エラーメッセージが表示されます。例えば、Microsoft SQL Server で Linux を選択する場合です。* **「注文に追加」**をクリックします。「ご清算」ページが表示されます。

  「ご清算」ページから構成ページへは、いずれかの「再構成」オプションをクリックすれば戻れます。
* 「拡張システム構成」セクションで、追加の構成オプションを指定します。詳しくは、『**[拡張システム構成](#adv-system-config)** 』を参照してください。

*   **「クラウド・サービスのご利用条件」**チェック・ボックスと**「サード・パーティー・サービスのご使用条件」**チェック・ボックスをクリックします。
*   支払情報を確認または入力し、**「注文の送信」**をクリックします。 プロビジョニングの注文番号を含む画面にリダイレクトされます。 この画面は、プロビジョニングの注文の受信を示すものでもあるため、印刷できます。

  この注文は、**「見積もりとして保存」**をクリックすれば、購入せずに保存しておくこともできます。

 一連の E メール (プロビジョニングの注文の確認応答、プロビジョニングの注文の承認と処理、およびプロビジョニングの完了) が管理者に送信されます。 プロビジョニング完了の E メールには、{{site.data.keyword.cloud_notm}} にログイン後の*「デバイスの詳細」*ページへのリンクが含まれています。 また、直接 {{site.data.keyword.slportal}}にログインすることもできます。

 ## 追加のサーバー構成オプション
 {: #addl-server-options}

 ベアメタル・サーバーのプロビジョニング時には、パブリック帯域幅、アップリンク・ポート速度、パブリック 2 次 IP アドレスをはじめ追加のオプションが選択可能です。表 1 で追加のオプションを説明します。


 | **フィールド** | **説明** |
 |-------------------|---------------|
 |サーバー・セキュリティー|Trusted Execution Technology (Intel TXT) など|
 |Software Guard Extensions|機密性の高いコードとデータのセキュリティー強化 (Intel SGX)。<br><br>『[Intel SGX を使用したベアメタル・サーバーのプロビジョニング](../bare-metal/bare-metal-provision-SGX.html)』を参照してください。|
 |RAM|サーバーのニーズに合った RAM のレベルを選択します。|
 |オペレーティング・システム |CentOS、FreeBSD、Microsoft、Red Hat、Ubuntu、他から選択します。|
 |ハード・ディスク |ユーザー・インターフェースで、OS の選択に基づいてフィールドに事前入力することでハード・ディスクをセットアップするツールを使用します。<br><br> Intel Optane SSD ドライブを使用するよう選択することもできます。『[Intel Optane SSD DC P4800X のプロビジョニング (Provisioning an Intel Optane SSD DC P4800X)](../bare-metal/bm-provision_ssd.html)』を参照してください。
 |パブリック帯域幅 |1 カ月間にパブリック・インターフェースを介して転送できるデータの量を決定します。 テスト環境ではインストール・データをこのインターフェースを使用して転送する必要がありますが、その環境では、値を初期転送データ量より大きくなるよう調整する必要があります。初期データ・ロードをいずれかの {{site.data.keyword.cloud_notm}} データ・センターに移送するには、[{{site.data.keyword.cloud_notm}}コンテンツ配信ネットワーク](https://www.ibm.com/cloud/cdn)を検討してください。任意の {{site.data.keyword.baremetal_short}} を、定額の (無制限の) 帯域幅を組み込むようにアップグレードできます。 すべての定額のデバイスは、プライベートの専用ポートに配置されます。|
 |アップリンク・ポート速度 |内部インターフェースおよび外部インターフェースの速度を決定します。 |
 |パブリック 2 次 IP アドレス |サーバーに割り当てる IP アドレスを追加します。シナリオによっては、追加 IP アドレスをサーバーに割り当てる必要が生じることがあります。 追加の IPv4 アドレスは、1、2、4、8、16、または 32 の数量で入手できます。 |
 |プライマリー IPv6 アドレス |サーバーの外部インターフェースに加え、内部インターフェースに割り当てられます。 |
 |パブリック静的 IPV6 アドレス |/64 ブロックから追加の IPv6 アドレスを割り当てます。 |
 |オペレーティング・システムのアドオン|オプションを選択します。例えば、VMware、バックアップ・ソリューション、コントロール・パネル、データベース、ハードウェアとソフトウェアのファイアウォール、アンチウィルスとスパイウェアからの保護、侵入検出と保護などがあります。<br><br>お客様の企業セキュリティー部門が {{site.data.keyword.cloud_notm}} サポートと連携して、これらのオプションの詳細について話し合うようにすることを強くお勧めします。
 |Evault |サーバー間でバックアップを複製するために、サーバーにインストールできるエージェント・ベースのバックアップ・ツール。 |
 |サービス・アドオン|モニタリング、自動応答、保険などのサービス・アドオンを選択します。|

## 拡張システム構成
{: #adv-system-config}

プロビジョニング・プロセスに必要なフィールドは、**「拡張システム構成」**の下にあるものがすべてです。

| **フィールド** | **説明** |
|---|---|
|ホスト名 | サーバーの永続的または一時的な名前。例えば _server1_ など。**注**: SAP 認定サーバーをプロビジョニングする場合、SAP ホスト名は最大 13 文字の英数字で構成する必要があります。SAP ホスト名について詳しくは、「[SAP Notes 611361](https://launchpad.support.sap.com/#/notes/2611361)」、および「[129997](https://launchpad.support.sap.com/#/notes/129997)」を参照してください。SAP S ユーザー ID が必要です。|
|ドメイン | インターネット・ドメイン名と競合しないサブドメイン名 (例えば、_test.acme.com_)。 |
| VLAN の選択 | 既に少なくともサーバーを 1 台は注文済みなので、ご使用のアカウントに VLAN があれば、新規サーバーをその VLAN に追加できます。|
|(プロビジョニング・スクリプト) |サーバーのプロビジョニング後に特定のステップを自動化できるようにするスクリプトを指定できます。|
| SSH 鍵 | SSH 鍵の公開鍵を指定できます。これにより、サーバーのプロビジョン後にサーバーにログインできるようになります。|
{: caption="表 2. 拡張システム構成" caption-side="top"}

 詳しくは、{{site.data.keyword.cloud_notm}} サポートにお問い合わせください。

 **注:** コンピュート・デバイスと併せて、2 次サブネットを注文することができます。ただし 2 次サブネットを注文した場合、コンピュート・デバイスがレクラメーション処理されると、そのサブネットもレクラメーション処理されます。2 次サブネットを、コンピュート・デバイスのアドオン・オプションとして注文するのではなく、独自に注文すれば、その 2 次サブネットは、自分で明示的にキャンセルするまで保持できます。コンピュート・デバイスがレクラメーション処理されても、不注意で IP アドレスを失うことがないように、このような違いがあることを覚えておいてください。
